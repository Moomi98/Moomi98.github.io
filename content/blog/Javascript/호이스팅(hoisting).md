## 🔍 호이스팅(hoisting)이란?

호이스팅이란 자바스크립트가 코드를 실행하기 전 변수 및 함수의 선언이 **해당 스코프의 최상단으로 끌어 올려진 것 같은 현상**을 말한다.

**실제로 최상단으로 끌어 올려지는 것이 아니라** 자바스크립트 엔진이 코드를 실행 할 때 모든 식별자를 스캔하면서 변수/함수들을 특정 객체에 담아놓고 시작한다.

이 개념에 대해서는 추후 작성하도록 하겠다.

위에서 언급했든 호이스팅의 종류에는 크게 변수 호이스팅과 함수 호이스팅이 있다.

## 📗 변수 호이스팅

변수의 선언 종류에는 크게 var, let, const 로 나뉘어진다.

var로 선언된 변수에만 변수 호이스팅이 발생하고 let, const로 선언된 변수에는 호이스팅이 발생하지 않는다.

var로 선언된 변수는 런타임 이전에 자바스크립트 엔진에 의해 암묵적으로 **"선언 단계"**와 **"초기화 단계"**가 한번에 실행된다.

즉, 선언 단계에서 스코프에 변수 식별자를 등록해 자바스크립트 엔진에 변수의 존재를 알린다. 그 후 초기화 단계에서 `undefined`로 할당한다.
따라서 변수 선언문 이전에 변수에 접근 시 참조 에러가 발생하지 않는다.

```javascript
console.log(foo) // undefined

var foo = 'hello'

console.log(foo) // hello
```

반면 let, const로 선언된 변수는 **"선언 단계"**와 **"초기화 단계"**가 분리되어 이루어 진다.

즉, 런타인 이전에 자바스크립트 엔진에 의해 암묵적으로 선언 단계가 먼저 실행되지만 초기화 단계는 변수 선언문에 도달했을 때 실행된다.

따라서 변수 선언문 이전에 변수에 접근 시 참조 에러가 발생한다.

```javascript
console.log(foo) // Reference Error!

let foo

console.log(foo) // undefined

foo = 'hello'

console.log(foo) //hello
```

위 예제에서 보면 코드 실행 후 foo 변수가 선언 되기 전에 foo를 참조하면 `Reference Error`가 발생했다. 이처럼 스코프의 시작 지점부터
초기화 시작 지점까지 변수를 참조할 수 없는 구간을 **일시적 사각지대**(Temporal Dead Zone) 이라 한다.

## 🎃 함수 호이스팅

함수의 정의 종류에는 함수 선언문과 함수 표현식이 있다.

```javascript
function add(x, y) {
  // 함수 선언문
  return x + y
}

const add = () => {
  // 함수 표현식
  return x + y
}
```

이 중 **함수 선언문**과 **var로 선언된 함수 표현식**에만 호이스팅이 적용된다.

여기서 변수 호이스팅과의 미묘한 차이가 존재하는데, var로 선언된 변수는 **런타임 이전**에`undefined`로 초기화되고 var로 선언된 함수 표현식은 **런타임 중**에 함수 객체로 초기화가 된다.

함수 호이스팅은 함수를 호출하기 전에 함수를 반드시 선언해야 된다는 당연한 규칙을 무시한다. 따라서 JSON의 창시자인 더글라스 크락포드는 함수 선언문 대신 **함수 표현식**을 사용할 것을 권장하고 있다.

## 🎈 마무리

호이스팅과 나중에 작성할 스코프 같은 개념들은 실행 컨텍스트과 클로저의 중요한 기본 개념이 된다.

자바스크립트라는 언어는 정말 양파같은 언어인 것 같다.😄
