## 🍪 쿠키

쿠키는 웹 브라우저에 저장되는 key,value 형태의 작은 파일이다.

쿠키는 다음과 같은 특징을 지니고 있다.

- **서버로 요청 시 항상 쿠키가 같이 전송**된다.

- 이에 따라 쿠키의 양이 크거나 많을 수록 서버에 부담이 갈 수 있다.

- 쿠키는 한 번에 약 4kb의 데이터를 저장할 수 있다.

- 대부분의 브라우저가 지원한다.

- **사용자가 임의로 수정 및 삭제가 가능**하므로, 지워도 상관없는 데이터를 넣는것이 좋다.

## 📦 웹 스토리지

웹 스토리지는 HTML5부터 지원하는 저장소의 형태이다.

크게 Local Storage와 Session Storage가 있다.

쿠키와 같이 key,value 형태로 데이터를 저장하며 다음과 같은 특징이 있다.

- 쿠키와 달리 **서버에 데이터가 매번 전송되지 않는다**.

- 약 5MB의 데이터를 저장할 수 있으며 유효기간이 존재하지 않는다.

- HTML5를 지원하지 않는 브라우저에서는 사용이 제한된다.

### Local Storage

- 브라우저가 종료되도 데이터는 영구적으로 저장된다.

- 도메인 별로 로컬 스토리지가 생성되며 서로 다른 브라우저 탭이라도 같은 브라우저의 도메인이면 같은 로컬 스토리지에 접근이 가능하다.

- 자동 로그인과 같은 영구적 데이터를 사용하기에 좋다.

### Session Storage

- 탭, 윈도우 단위로 세션 스토리지가 생성되며 탭을 닫을 경우 데이터가 소멸된다.

- 또한 동일한 탭/윈도우라도 다른 도메인이라면 또 다른 세션 스토리지가 생성된다.

- 서로 다른 세션 스토리지는 서로 영향을 주지 않으며 독립적으로 동작한다.

- 잠시 동안 필요한 정보를 저장하기에 좋다. (ex. 입력 폼 저장, 일회성 로그인 등)
